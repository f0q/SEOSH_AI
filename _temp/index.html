Три ключевых понятия в контент-плане

В xlsx-файле три колонки могут выглядеть похоже, но описывают разные вещи. Вот чёткое разграничение:

Колонка B
Раздел
Навигационная группа страниц — где страница находится в меню сайта. Один раздел содержит несколько страниц.
Примеры: «Услуги», «Продукция под нанесение», «Блог», «Акции»
Колонка C
Тип страницы
Функциональный шаблон — что делает страница. Определяет структуру блоков, количество H2, целевой объём текста. Разные разделы могут использовать один тип.
Примеры: service_detail, blog_post, landing_page
Колонка P
Schema
Разметка Schema.org (JSON-LD) — что говорим Google об этой странице. Добавляется программистом в код. Влияет на rich snippets: цены, FAQs, звёздочки рейтинга.
Примеры: Service, Product, Article

Как работать с контент-планом
1
Открыть шаблоны и понять структуру каждого типа
Перед написанием текстов откройте интерактивные шаблоны и изучите схему блоков нужного типа страницы. Там указан порядок разделов, целевой объём каждого блока и примеры H2 с сайта конкурента i-maika.ru.
templates_viewer.html Схема блоков Объём текста
2
Взять черновик из xlsx и передать копирайтеру (в нашем случае - делаем сами)
В xlsx уже готовы черновики Title, Meta Description, H1 и четыре варианта H2 для каждой страницы. Черновики — стартовая точка, не финальный текст. Копирайтер дорабатывает их под реальную страницу. Ключевые слова в колонке Q можно использовать как ТЗ на оптимизацию.
content_plan_fackturaf.xlsx Лист «Контент-план» Колонки F, H, J, K–N
3
Проверить длины Title и Meta Description
Title — 50–65 символов, Meta Description — 140–160 символов. В колонках G и I xlsx уже посчитана длина черновика. Если копирайтер меняет текст — перепроверьте длину вручную. Удобный инструмент: просто вставьте текст в поле поиска Google и посмотрите, как выглядит сниппет.
Колонка G — длина Title Колонка I — длина Meta Desc
4
Добавить Schema-разметку на каждый тип страниц
В колонке P указан нужный тип Schema.org. Разметка добавляется программистом один раз как шаблон для типа страницы — не для каждой страницы отдельно. Например: все страницы service_detail получают Schema: Service, все blog_post — Article. Проверить корректность разметки: Rich Results Test.
Колонка P — Schema JSON-LD Rich Snippets
5
Обновлять статус в таблице и следить за индексацией
Колонка E «Статус» — для отметки прогресса («В работе», «Готово», «Опубликовано»). После публикации добавьте URL в Google Search Console → Проверка URL для ускоренной индексации. Первые результаты в поиске по коммерческим запросам — через 1–3 месяца, по блогу — через 2–4 месяца.
Колонка E — Статус Google Search Console Индексация URL
Важно об объёме текста Целевые объёмы взяты из анализа конкурента i-maika.ru. Для коммерческих страниц (услуги, лендинги) — 2 000–3 500 слов. Для блога — 1 000–1 500 слов. Больше текста само по себе не даёт позиций — важна польза для читателя и уникальность. Обязательна проверка уникальности (от 90%) и «тошноты» (не выше 8%) перед публикацией.
